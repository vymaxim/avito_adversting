Приложение реализовано на языке Python в связке  с PostgresQL
### Запуск приложения

```python3 manage.py```

## Метод получения списка объявлений


- Пагинация: на одной странице должно присутствовать 10 объявлений;
- Cортировки: по цене (возрастание/убывание) и по дате создания (возрастание/убывание);
- Поля в ответе: название объявления, ссылка на главное фото (первое в списке), цена.

Запрос осуществлен методом POST, в теле запроса передается JSON с параметрами:
   
    {"page": "x", "sort": "y"}

значение х:

    целое число
значение y:

    "price_asc": сортировка по возрастанию цены
    "price_desc": сортировка по убыванию цены
    "date_asc": сортировка по возрастанию даты создания
    "date_desc": сортировка по убыванию даты создания

## Метод получения конкретного объявления

- Обязательные поля в ответе: название объявления, цена, ссылка на главное фото;
- Опциональные поля (можно запросить, передав параметр fields): описание, ссылки на все фото.

Запрос осуществлен методом POST, в теле запроса передается JSON с параметрами:

    {"id": x, "fields": y}
    

значение х:

    целое число
значение y:

    True or False 

## Метод создания объявления

- Принимает все вышеперечисленные поля: название, описание, несколько ссылок на фотографии (сами фото загружать никуда не требуется), цена;
- Возвращает ID созданного объявления и код результата (ошибка или успех).

Запрос осуществлен методом POST, в теле запроса передается JSON вида:

    {       
    "name": ,
    "price": ,
    "description": ,
    "main_url": ,
    "url2": ,
    "url3": }
